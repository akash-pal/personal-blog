### **Redux: Three Principle**

Single Source of truth

*   One state tree inside Store
*   Predictability, maintainability
*   Universal apps (SSR — Server-Side Rendering)
*   Testing and Debugging

State is read-only

*   Derive properties from State.
*   Dispatch actions to change the State.
*   Immutable update patterns

Pure function update state

*   Pure functions are reducers
*   Reducers respond to action types
*   Reducers return a new state

### **Redux Core Concepts**

#### Single State Tree

1.  Plain Javascript Object

2\. Composed by reducers

const state = {  
 todos:\[\]  
}

#### **Actions**

1.  Two properties:

*   **type:** string, describes an event
*   **payload:** optional data

2\. Dispatch actions to reducers

const action = {  
 type:'ADD\_TODO',  
  payload:{  
   label:'Eat Pizza',  
    complete:false  
  }  
}

#### Reducers

1.  Pure functions
2.  Given dispatch actions:

*   Responds to action.type
*   Access to action.payload
*   Composes new state
*   Returns new state

function reducers(state,action){  
 switch(action.type){  
   case 'ADD\_TODO':{  
     const todo = action.payload;  
      const todos = \[...state.todos,todo\]  
      return {todos};  
    }  
  }  
  return state;  
}

New state

const state = {  
  todos:\[{  
    label:'Eat Pizza',  
    complete:false  
  }\]  
}

#### Store

1.  State Container
2.  Containers interact with the store
3.  Store invokes Reducers with previous State and Action
4.  Reducers compose a new state
5.  The store is updated, notifies subscribers
